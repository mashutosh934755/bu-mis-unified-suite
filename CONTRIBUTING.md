# Contributing

Contributions are welcome.

## Development Rules

1. Create a new branch.
2. Keep each module independent.
3. Do not commit secrets.
4. Follow WordPress coding and security practices.
5. Add documentation for any new module.
6. Test on a staging WordPress installation.

## Suggested Branch Names

```text
feature/dspace-live-count
feature/ojs-dashboard
fix/public-dashboard-layout
docs/readme-update
```

## Pull Request Checklist

- [ ] No secrets committed
- [ ] PHP syntax checked
- [ ] WordPress cache cleared during testing
- [ ] Screenshots added if UI changed
- [ ] Documentation updated
